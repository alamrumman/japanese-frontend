# Overview

This repository is **not** a traditional Product Requirement Document (PRD).

It is a complete **Design & Engineering Specification** that defines the visual language, interaction patterns, motion principles, accessibility standards, engineering architecture, and implementation details required to build a premium Japanese restaurant website.

The final product should feel handcrafted by an award-winning design studio rather than generated from a template or AI.

Every page, interaction, animation and transition must communicate craftsmanship, restraint and elegance.

# Motion Philosophy

Every animation must answer one question.

Why is this moving?

Animation exists to

Guide Attention

Reveal Information

Support Storytelling

Improve Usability

Animation must never

Decorate

Delay Interaction

Compete with Content

If removing an animation does not reduce clarity—

remove it.

# AI Self Review

Before completing any page ask

Does this resemble an Awwwards quality website?

Would this still look premium without animations?

Does typography dominate over effects?

Is photography the visual focus?

Does every section have its own identity?

Is whitespace doing the heavy lifting?

Does the layout feel handcrafted?

Would a senior frontend engineer believe this was professionally designed?

If any answer is "No"

Redesign before continuing.
